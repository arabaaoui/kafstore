Kafstore

Kafka Keystore Generator - Automated SSL Configuration

Secure Server-Side Processing
What does this tool do?
Kafstore converts your certificates into ready-to-use Java keystores (JKS/PKCS12) for Kafka mTLS authentication. Just 3 files needed - No more manual keytool commands!

Upload Certificate Files

Upload only 3 files from your certificate authority. File extensions don't matter!

CA Chain ? Complete certificate authority chain. Used to create the truststore for server validation. CA_chain Usually named: CA_chain, ca-bundle, chain.pem
Client Certificate ? Your client certificate file. This is the certificate issued to you by the CA. cert Usually named: cert, certificate, client.crt, client.pem
Private Key ? Your private key corresponding to the client certificate. Keep this secure! key Usually named: key, private-key, client.key

Configuration

Configure your keystore parameters. You can use different passwords for truststore and keystore.

Generate Keystores

Generate your keystores and download them as a ZIP file.

truststore.jks
Java KeyStore with CA certificates for server validation
keystore.jks
Java KeyStore with your client certificate
keystore.p12
PKCS12 format keystore (alternative to JKS)
client-ssl.properties
Ready-to-use Kafka configuration file

Test Kafka Connection

Test your generated keystores with a real Kafka broker to ensure everything works.

Manual Commands ? Use these commands with your generated keystores to interact with Kafka from the command line.

⚠️ Important: Before using the properties file, update the paths inside it to absolute paths:
ssl.truststore.location=/absolute/path/to/truststore.jks
ssl.keystore.location=/absolute/path/to/keystore.jks
# List topics ./kafka-topics.sh --list \ --command-config client-ssl.properties \ --bootstrap-server <bootstrap>:443 # Consume messages ./kafka-console-consumer.sh \ --topic <topic-name> \ --from-beginning --max-messages 1 \ --consumer.config client-ssl.properties \ --bootstrap-server <bootstrap>:443
